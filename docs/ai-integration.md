# AI Integration — Claude Plan Generation

## Overview

Training plans are generated by calling the Anthropic Claude API from the `createPlan` server action (`app/src/app/actions.ts`). The user's profile (goals, grade, age, equipment, schedule) is sent as a structured prompt. Claude returns a JSON object representing the full plan, which is parsed and saved to PostgreSQL.

## Model

`claude-3-5-haiku-20241022` — fast and cost-effective for structured JSON generation. Upgrade to `claude-opus-4-7` for higher-quality plans if latency/cost is acceptable.

## Prompt structure

The system prompt instructs Claude to act as an expert climbing coach and return valid JSON only. The user message provides the profile fields.

Expected response shape:
```json
{
  "weeks": [
    {
      "weekNum": 1,
      "theme": "Foundation & Technique",
      "days": [
        {
          "dayNum": 1,
          "dayName": "Monday",
          "focus": "Finger Strength",
          "isRest": false,
          "sessions": [
            {
              "name": "Warm-Up",
              "description": "Prepare joints and tendons",
              "duration": 20,
              "exercises": [
                {
                  "name": "Easy traversing",
                  "sets": null,
                  "reps": null,
                  "duration": "10 min",
                  "rest": null,
                  "notes": "Focus on footwork",
                  "order": 0
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

## API key

Set `ANTHROPIC_API_KEY` in:
- `app/.env` — local development
- `docker-compose.yml` web service environment — Docker deployment

The key should start with `sk-ant-` for direct Anthropic access. OpenRouter keys (`sk-or-v1-`) also work if the base URL is set to `https://openrouter.ai/api/v1` and the model name is prefixed accordingly.

## SDK usage

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

const message = await client.messages.create({
  model: "claude-3-5-haiku-20241022",
  max_tokens: 8192,
  system: SYSTEM_PROMPT,
  messages: [{ role: "user", content: userPrompt }],
});
```

## Error handling

If the Claude API call fails or returns malformed JSON, `createPlan` should catch the error and redirect back to onboarding with an error message rather than creating a partial plan in the DB.

## Cost estimate (approximate)

| Plan length | Estimated tokens | Estimated cost (Haiku) |
|---|---|---|
| 4 weeks | ~3,000 output | ~$0.003 |
| 8 weeks | ~5,500 output | ~$0.006 |
| 12 weeks | ~8,000 output | ~$0.009 |
| 16 weeks | ~10,500 output | ~$0.012 |

At 1,000 plan generations/day, a 12-week plan costs ~$9/day in AI API calls.

## Future improvements

- **Streaming** — stream the JSON response and show a progress indicator while the plan is being generated (Next.js supports streaming Server Actions)
- **Plan regeneration** — allow users to regenerate a plan with different parameters without losing their logged workouts
- **Chat refinement** — after plan generation, let the user ask follow-up questions to adjust the plan
- **Progress-aware replanning** — feed logged workout data back to Claude to adapt upcoming weeks based on performance
